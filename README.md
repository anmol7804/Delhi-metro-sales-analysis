# Delhi-metro-sales-analysis
🚇 Delhi Metro Data Analysis
📌 Project Summary

This project analyzes Delhi Metro trip data to extract meaningful insights about passenger behavior, revenue generation, and operational efficiency.
It demonstrates end-to-end data analysis including data cleaning, exploration, and visualization using Python.

🎯 Business Problem

Understanding metro usage patterns is important to:

Improve passenger experience

Identify high-demand stations

Optimize pricing and operations

Increase profitability

📊 Dataset Details

The dataset includes trip-level information such as:

Trip_id, Date

From_Station, To_Station

Distance_km

Fare, Cost_per_passenger

Passengers, Ticket_type

Profit_per_passenger, Profit

Remarks

🧹 Data Preprocessing

Cleaned inconsistent text and removed extra spaces

Handled missing values (NaN)

Standardized column names

Ensured correct data types (Date, numeric columns)

📈 Exploratory Data Analysis
Key Analyses Performed:

Passenger distribution by ticket type

Busiest stations (incoming + outgoing traffic)

Most profitable routes

Daily revenue and profit trends

Fare distribution and outlier detection

Relationship between distance and fare

📊 Visualizations

📊 Bar Chart → Passengers by Ticket Type

📈 Line Chart → Daily Revenue vs Profit

🔵 Scatter Plot → Distance vs Fare

📦 Box Plot → Fare Distribution

📊 Bar Chart → Top Profitable Routes

🚉 Bar Chart → Busiest Stations

🔍 Key Insights

High passenger traffic is concentrated in a few major stations

Card-based tickets are preferred by most commuters

Certain routes generate significantly higher profit

Fare structure is consistent and distance-based

Average trip fare is approximately ₹100

✅ Conclusion

The analysis highlights clear patterns in passenger flow, revenue, and profitability.
These insights can help in better planning, pricing strategies, and improving metro operations.
